Project Overview

Objective

Develop a systematic, reproducible methodology for predicting wind turbine failures 24 hours in advance using SCADA sensor data, enabling proactive maintenance and cost savings.

Approach

Built farm-specific models using Random Forest classification with window aggregation features (mean, std, trend). Validated transferability across three independent wind farms with different turbine models.

Pivot

Semantic sensor mapping enables unified multi-farm model, combining training data from heterogeneous sources to improve performance on limited-data scenarios (+17% on Farm B).

Key Findings

Consistent 67% Benchmark

Farm A (development) and Farm C (validation) both achieved 67% recall despite different environments and 12x sensor difference (81 vs 952), demonstrating methodology transferability.

Perfect Validation Precision

100% precision on Farms B & C (zero false alarms) maintains operator trust. Critical for production deployment where false positives cost $5K-$10K per unnecessary inspection.

Universal Failure Patterns

Mechanical degradation (rotor speed variability, gearbox instability) and pitch control issues are strongest predictors across all turbine models and environments.

Multi-Domain Trade-offs

Unified model improves limited-data performance (Farm B: 33% → 50%, +17%) but sacrifices peak performance on well-instrumented farms (-7 to -9%). Trade-off accepted for consistent baseline.

Technical Highlights

Data Processing

  • 2.5M+ SCADA records (10-min intervals)
  • Validated temporal patterns (7-day vs sudden failures)
  • Critical insight: status==0 AND power>0.1 for true normal operation
  • ±30 day exclusion zones around failures

Feature Engineering

  • Window aggregation: mean, std, linear trend over 24h
  • Scales from 81 to 952 sensors automatically
  • Farm-specific: 243-2,856 features generated
  • Unified: 78 features across 16 semantic categories

Feature Selection

  • Cohen's d for discriminative power (threshold: 0.8-1.0)
  • Iterative correlation removal (r < 0.9)
  • Result: 15-24 non-redundant features per farm
  • Massive redundancy discovered: 214 pairs r>0.9

Model Evaluation

  • Random Forest (100 trees, max_depth=5, balanced)
  • Leave-One-Out CV (appropriate for n=6-27 failures)
  • Per-event probability tracking
  • Validation-first methodology (data quality → features → selection → model)
